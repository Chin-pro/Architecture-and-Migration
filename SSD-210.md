| 測試類型 | 工具 | 適合測什麼 |
| --- | --- | --- |
| Controller Test | MockMvc | request mapping、status code、validation |
| Integration Test | SpringBootTest + MinIO Testcontainer | Controller + Service + MinIO |
| API-level E2E Test | SpringBootTest RANDOM_PORT + REST Assured + MinIO Testcontainer | 真 HTTP 上傳 / 下載流程 |
| UI E2E Test | Playwright | 前端畫面點擊、上傳、下載 |

下面我直接給你一套**可以落地到 Spring Boot 2 專案的做法**。你可以把它理解成四層測試，但你這次臨時報案最優先做的是：

```
第一優先：Controller Test
目的：確認 API 接口格式、multipart 參數、status code 沒壞。

第二優先：API-level E2E Test
目的：真的啟動 Spring Boot，真的用 HTTP 上傳 / 下載，真的接 MinIO。
```

Spring Boot 官方文件說 `@WebMvcTest` 會自動設定 MockMvc，MockMvc 可以不用啟動完整 HTTP server 就快速測 Controller；而 `@SpringBootTest(webEnvironment = RANDOM_PORT)` 則適合啟動真實 server 測 HTTP 流程。(Home)

MinIO 這種外部服務可以用 Testcontainers 啟動暫時 container，Testcontainers 官方也提供 GenericContainer 與 MinIO container 的使用方式。(Testcontainers for Java)

# 2. 建議加一個 e2e Maven profile

不要讓 E2E Test 進入每次 `mvn test` 必跑。公司內網 CI/CD 未必支援 Docker，所以先用 profile 隔離。

```xml
<profiles>
    <profile>
        <id>e2e</id>
        <build>
            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-surefire-plugin</artifactId>
                    <configuration>
                        <includes>
                            <include>**/*E2ETest.java</include>
                            <include>**/*IntegrationTest.java</include>
                        </includes>
                    </configuration>
                </plugin>
            </plugins>
        </build>
    </profile>
</profiles>
```

平常開發跑：

```bash
mvn test
```

跑 E2E 時才用：

```bash
mvn test -Pe2e
```

如果公司已經用 `maven-failsafe-plugin` 管 integration test，那更標準是讓 `*IT.java` 或 `*E2ETest.java` 在 `mvn verify -Pe2e` 跑。但臨時報案先不用把 Maven 結構做太大。

# 4. 你要先確認 production code 讀哪幾個 property

你要去專案裡搜尋：

```java
@Value("${")
```

或：

```java
@ConfigurationProperties
```

你要找出 MinIO 設定實際名稱。

可能是這種：

```java
@Value("${minio.endpoint}")
private String endpoint;

@Value("${minio.access-key}")
private String accessKey;

@Value("${minio.secret-key}")
private String secretKey;

@Value("${minio.bucket-name}")
private String bucketName;
```

也可能是：

```java
@ConfigurationProperties(prefix = "storage.minio")
```

如果是第二種，你測試要塞的 property 就不是：

```java
minio.endpoint
```

而是：

```java
storage.minio.endpoint
storage.minio.access-key
storage.minio.secret-key
storage.minio.bucket-name
```

**這是最容易錯的地方。**

測試明明啟動了 MinIO，但 Spring Boot App 讀到的是空值或舊設定，最後你會誤以為 Testcontainers 壞掉。

# 5. 第一層：Controller Test，用 MockMvc 測接口格式

這層不測 MinIO。

它只測：

```
POST /api/vendors/items/upload 是否收 multipart file
siteId 是否必填
沒上傳 file 是否回 400
service 是否被正確呼叫
下載 API 是否回正確 header
```

Spring Boot 官方文件說 `@WebMvcTest` 會自動設定 MockMvc，適合快速測 MVC controller，不需要啟動完整 HTTP server。(Home)

假設你的 Controller 長得像這樣：

```java
@PostMapping("/api/vendors/items/upload")
public ResponseEntity<?> upload(
        @RequestParam("siteId") String siteId,
        @RequestParam("file") MultipartFile file
) {
    vendorFileService.upload(siteId, file);
    return ResponseEntity.ok().body(Map.of("message", "success"));
}
```

那 Controller Test 可以這樣寫：

```java
package com.company.project.controller;

import com.company.project.service.VendorFileService;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.boot.test.mock.mockito.MockBean;
import org.springframework.mock.web.MockMultipartFile;
import org.springframework.test.web.servlet.MockMvc;

import static org.mockito.ArgumentMatchers.any;
import static org.mockito.ArgumentMatchers.eq;
import static org.mockito.Mockito.verify;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.multipart;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;

@WebMvcTest(VendorFileController.class)
class VendorFileControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private VendorFileService vendorFileService;

    @Test
    void upload_shouldAcceptMultipartFileAndSiteId() throws Exception {
        MockMultipartFile file = new MockMultipartFile(
                "file",
                "vendor-upload.xlsx",
                "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
                "fake excel content".getBytes()
        );

        mockMvc.perform(
                        multipart("/api/vendors/items/upload")
                                .file(file)
                                .param("siteId", "CY")
                )
                .andExpect(status().isOk());

        verify(vendorFileService).upload(eq("CY"), any());
    }

    @Test
    void upload_shouldReturnBadRequest_whenFileIsMissing() throws Exception {
        mockMvc.perform(
                        multipart("/api/vendors/items/upload")
                                .param("siteId", "CY")
                )
                .andExpect(status().isBadRequest());
    }
}
```

這支測試的價值是：

你以後改 Controller、改參數名稱、改 endpoint，如果不小心破壞前端或 Thunder Client 呼叫格式，這層會先爆。

# 6. 建立共用的 MinIO 測試支援類

第二層和第三層都會用到 MinIO，所以建議抽一個測試用 support class：

```
src/test/java/com/company/project/integration/support/MinioTestSupport.java
```

```java
package com.company.project.integration.support;

import io.minio.BucketExistsArgs;
import io.minio.MakeBucketArgs;
import io.minio.MinioClient;
import io.minio.PutObjectArgs;
import org.springframework.core.io.ClassPathResource;
import org.testcontainers.containers.GenericContainer;
import org.testcontainers.containers.wait.strategy.Wait;
import org.testcontainers.utility.DockerImageName;

import java.io.InputStream;

public abstract class MinioTestSupport {

    protected static final String MINIO_USER = "minioadmin";
    protected static final String MINIO_PASSWORD = "minioadmin";
    protected static final String BUCKET_NAME = "stationery-e2e-test";

    protected static final String TEMPLATE_OBJECT_PATH =
            "templates/vendor-upload-template.xlsx";

    protected static final GenericContainer<?> MINIO_CONTAINER =
            new GenericContainer<>(
                    DockerImageName.parse("minio/minio:RELEASE.2023-09-04T19-57-37Z")
            )
                    .withEnv("MINIO_ROOT_USER", MINIO_USER)
                    .withEnv("MINIO_ROOT_PASSWORD", MINIO_PASSWORD)
                    .withCommand("server", "/data")
                    .withExposedPorts(9000)
                    .waitingFor(
                            Wait.forHttp("/minio/health/ready")
                                    .forPort(9000)
                    );

    protected static MinioClient minioClient;

    protected static String getMinioEndpoint() {
        return "http://" + MINIO_CONTAINER.getHost()
                + ":" + MINIO_CONTAINER.getMappedPort(9000);
    }

    protected static void initializeMinio() throws Exception {
        minioClient = MinioClient.builder()
                .endpoint(getMinioEndpoint())
                .credentials(MINIO_USER, MINIO_PASSWORD)
                .build();

        boolean bucketExists = minioClient.bucketExists(
                BucketExistsArgs.builder()
                        .bucket(BUCKET_NAME)
                        .build()
        );

        if (!bucketExists) {
            minioClient.makeBucket(
                    MakeBucketArgs.builder()
                            .bucket(BUCKET_NAME)
                            .build()
            );
        }

        uploadTemplateToMinio();
    }

    private static void uploadTemplateToMinio() throws Exception {
        ClassPathResource templateResource =
                new ClassPathResource("e2e/vendor-template.xlsx");

        try (InputStream inputStream = templateResource.getInputStream()) {
            minioClient.putObject(
                    PutObjectArgs.builder()
                            .bucket(BUCKET_NAME)
                            .object(TEMPLATE_OBJECT_PATH)
                            .stream(inputStream, templateResource.contentLength(), -1)
                            .contentType("application/vnd.openxmlformats-officedocument.spreadsheetml.sheet")
                            .build()
            );
        }
    }
}
```

這裡用的是 `GenericContainer`，因為 Testcontainers 官方說 GenericContainer 最有彈性，可以用幾乎任何 container image 當測試依賴。(Testcontainers for Java)

雖然 Testcontainers 有 MinIO module，但在 Spring Boot 2、公司舊專案、臨時報案情境，我會先用 GenericContainer，降低版本衝突風險。MinIO module 官方範例則是 `MinIOContainer` 搭配 MinIO Java client。(Testcontainers for Java)

# 7. 第二層：Integration Test，測 Service + MinIO

這層可以不走真實 HTTP。

它測的是：

```
Service 是否能把檔案存到 MinIO
Service 是否能從 MinIO 讀 template
vendor name 重複時是否用 siteId 正確判斷
```

```java
package com.company.project.integration;

import com.company.project.integration.support.MinioTestSupport;
import com.company.project.service.VendorFileService;
import io.minio.StatObjectArgs;
import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.Tag;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.mock.web.MockMultipartFile;
import org.springframework.test.context.ActiveProfiles;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;

import java.io.InputStream;

import static org.assertj.core.api.Assertions.assertThat;

@Tag("e2e")
@Testcontainers
@ActiveProfiles("e2e")
@SpringBootTest
class VendorFileIntegrationTest extends MinioTestSupport {

    @Autowired
    private VendorFileService vendorFileService;

    @Container
    static final org.testcontainers.containers.GenericContainer<?> minio =
            MINIO_CONTAINER;

    @DynamicPropertySource
    static void registerMinioProperties(DynamicPropertyRegistry registry) {
        registry.add("minio.endpoint", MinioTestSupport::getMinioEndpoint);
        registry.add("minio.access-key", () -> MINIO_USER);
        registry.add("minio.secret-key", () -> MINIO_PASSWORD);
        registry.add("minio.bucket-name", () -> BUCKET_NAME);
    }

    @BeforeAll
    static void beforeAll() throws Exception {
        initializeMinio();
    }

    @Test
    void upload_shouldStoreFileIntoMinio() throws Exception {
        byte[] fileBytes = "fake excel content".getBytes();

        MockMultipartFile file = new MockMultipartFile(
                "file",
                "vendor-upload.xlsx",
                "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
                fileBytes
        );

        vendorFileService.upload("CY", file);

        // 這裡 object path 要改成你 service 實際存放的位置
        minioClient.statObject(
                StatObjectArgs.builder()
                        .bucket(BUCKET_NAME)
                        .object("uploads/CY/vendor-upload.xlsx")
                        .build()
        );
    }

    @Test
    void downloadTemplate_shouldReturnBytes() throws Exception {
        byte[] downloadedBytes = vendorFileService.downloadTemplate("CY");

        assertThat(downloadedBytes).isNotEmpty();
    }
}
```

注意：這段的 object path：

```java
"uploads/CY/vendor-upload.xlsx"
```

一定要改成你公司專案實際邏輯。

如果 service 會用 UUID 產生檔名，你就不要硬寫固定 object path，而是從 DB 或 response 拿 object path 再驗證 MinIO。

# 8. 第三層：API-level E2E Test，真 HTTP 上傳 / 下載

這層是你這次最有價值的測試。

它會真的啟動 Spring Boot server，真的打 HTTP request，真的接 MinIO。

Spring Boot 官方文件示範 `@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)` 可以啟動 random port server；文件也有搭配 RestAssured 對 random port 發 request 的範例。(Home)

```java
package com.company.project.e2e;

import com.company.project.integration.support.MinioTestSupport;
import io.minio.StatObjectArgs;
import io.restassured.RestAssured;
import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.Tag;
import org.junit.jupiter.api.Test;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.boot.web.server.LocalServerPort;
import org.springframework.core.io.ClassPathResource;
import org.springframework.test.context.ActiveProfiles;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;

import java.io.File;

import static io.restassured.RestAssured.given;
import static org.assertj.core.api.Assertions.assertThat;
import static org.hamcrest.Matchers.containsString;

@Tag("e2e")
@Testcontainers
@ActiveProfiles("e2e")
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class VendorFileUploadDownloadE2ETest extends MinioTestSupport {

    @LocalServerPort
    private int port;

    @Container
    static final org.testcontainers.containers.GenericContainer<?> minio =
            MINIO_CONTAINER;

    @DynamicPropertySource
    static void registerMinioProperties(DynamicPropertyRegistry registry) {
        registry.add("minio.endpoint", MinioTestSupport::getMinioEndpoint);
        registry.add("minio.access-key", () -> MINIO_USER);
        registry.add("minio.secret-key", () -> MINIO_PASSWORD);
        registry.add("minio.bucket-name", () -> BUCKET_NAME);
    }

    @BeforeAll
    static void beforeAll() throws Exception {
        initializeMinio();
    }

    @Test
    void uploadVendorItemFile_shouldUploadSuccessfully() throws Exception {
        RestAssured.port = port;

        File uploadFile = new ClassPathResource(
                "e2e/vendor-upload-duplicate-name-cy.xlsx"
        ).getFile();

        given()
                .multiPart(
                        "file",
                        uploadFile,
                        "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
                )
                .formParam("siteId", "CY")
                .when()
                .post("/api/vendors/items/upload")
                .then()
                .statusCode(200)
                .body("message", containsString("success"));

        // 如果你的上傳流程會保存原始檔，可加這段
        // object path 要改成你專案實際儲存邏輯
        minioClient.statObject(
                StatObjectArgs.builder()
                        .bucket(BUCKET_NAME)
                        .object("uploads/CY/vendor-upload-duplicate-name-cy.xlsx")
                        .build()
        );
    }

    @Test
    void downloadVendorTemplate_shouldReturnExcelFile() {
        RestAssured.port = port;

        byte[] downloadedBytes =
                given()
                        .queryParam("siteId", "CY")
                        .when()
                        .get("/api/vendors/items/template")
                        .then()
                        .statusCode(200)
                        .header("Content-Disposition", containsString("attachment"))
                        .extract()
                        .asByteArray();

        assertThat(downloadedBytes).isNotEmpty();
        assertThat(downloadedBytes.length).isGreaterThan(100);
    }
}
```

如果你的 Spring Boot 2 版本比較新，`@LocalServerPort` 可能要改成：

```java
import org.springframework.boot.test.web.server.LocalServerPort;
```

如果比較舊，通常是：

```java
import org.springframework.boot.web.server.LocalServerPort;
```

以 IDE 能 compile 的為準。

# 9. 如果下載 API 回傳 presigned URL

有些系統不會直接回傳檔案，而是回傳：

```json
{
  "downloadUrl": "http://..."
}
```

那 E2E Test 要變成兩段：

```java
@Test
void downloadVendorTemplate_shouldReturnUsablePresignedUrl() {
    RestAssured.port = port;

    String downloadUrl =
            given()
                    .queryParam("siteId", "CY")
                    .when()
                    .get("/api/vendors/items/template-url")
                    .then()
                    .statusCode(200)
                    .extract()
                    .path("downloadUrl");

    byte[] downloadedBytes =
            given()
                    .when()
                    .get(downloadUrl)
                    .then()
                    .statusCode(200)
                    .extract()
                    .asByteArray();

    assertThat(downloadedBytes).isNotEmpty();
}
```

不要只驗證 `downloadUrl` 不為空。

因為 URL 有值，不代表真的能下載。

# 12. 針對你這次 bug 的測試情境應該這樣設計

你的 bug 核心不是 MinIO。

你的 bug 核心是：

```
嘉義廠區新增後，
vendor name 與台南廠區重複，
原本 getMapName 只靠 vendor name 對應，
導致上傳 / 刪除時抓錯 vendor。
```

所以 E2E Test 的資料應該刻意準備這種狀況：

```

Tainan siteId = TN
Chiayi siteId = CY

site_id = TN, vendor_name = ABC供應商, vendor_code = TN001

```
Given:

When:

```java

```

如果 response 不回，就查 DB：

```java

SQL table 名稱要改成你公司實際的 table。

# 1. 你不應該讓所有測試都真的登入

你的測試要分成 5 層：

```
Level 1：Unit Test
不碰登入、不碰 TSSO、不碰 Keycloak。

Level 2：Controller / API Test
用假 user、假 header、假 session，避開正式登入。

Level 3：API-level E2E Test
真的啟動 Spring Boot、真的打 API、真的接 MinIO / DB，
但登入用 test profile bypass 或 mock auth。

Level 4：UI E2E Test
必要時才真的打開瀏覽器。
登入狀態用 Playwright storage state 重用，不要每支測試重新登入。

Level 5：True SSO Smoke Test
真的走 TSSO + Keycloak。
只做少數幾支，手動觸發、nightly 或 release 前跑。
```

你現在的錯誤是把 Level 3 的任務，硬做成 Level 5。

你只是要測：

```
上傳 Excel
下載 template
vendor name 重複時是否對應正確 site/vendor
MinIO 是否能讀寫
```

這些不應該每次都依賴正式 TSSO / Keycloak。

# 7. 具體實作方式四：真的拿 Keycloak token，但不走 UI 登入

如果你的後端 API 接受 Bearer Token，那你可以用 Keycloak token endpoint 直接取得 token，不需要打開瀏覽器。

Keycloak 文件中，authorization code flow 最後會用 authorization code 搭配 client credentials 去取得 Access Token、Refresh Token、ID Token。

REST Assured 也支援 OAuth2 bearer token，例如 `given().auth().oauth2(token)` 這類寫法。

測試概念：

```
測試開始
  ↓
向 Keycloak token endpoint 拿 access token
  ↓
REST Assured 呼叫 API 時帶 Authorization: Bearer xxx
  ↓
後端驗證 token
  ↓
執行上傳 / 下載測試
```

Java 範例：

```
privateStringgetAccessToken() {
returngiven()
.contentType("application/x-www-form-urlencoded")
.formParam("grant_type","client_credentials")
.formParam("client_id","test-client")
.formParam("client_secret","test-secret")
.when()
.post("https://keycloak-test.company.com/realms/test/protocol/openid-connect/token")
.then()
.statusCode(200)
.extract()
.path("access_token");
}
```

使用 token 呼叫 API：

```
Stringtoken=getAccessToken();

given()
.auth().oauth2(token)
.multiPart("file",uploadFile)
.formParam("siteId","CY")
.when()
.post("/api/vendors/items/upload")
.then()
.statusCode(200);
```

但你目前說公司「沒有 Spring Security」，所以後端不一定吃 Bearer Token。

如果後端只是靠 TSSO session / cookie / 內網模組，那這條路可能不適用。

# 8. UI E2E：不要每次登入，用 Playwright 保存登入狀態

如果你真的要測 UI，例如真的打開瀏覽器點上傳，那可以用 Playwright 的 storage state。

Playwright 官方文件說，Web app 的登入狀態通常存在 cookies、local storage 或 IndexedDB；Playwright 可以用 `BrowserContext.storageState()` 取得已登入狀態，之後建立新的 context 時重用這些狀態。

概念是：

```
第一次：
手動或自動登入一次
↓
把登入後的 cookie/localStorage 存成 auth-state.json

之後：
測試直接載入 auth-state.json
↓
打開頁面時已經是登入狀態
↓
不用每支測試重新登入 Keycloak
```

官方也建議把 authentication state 放在類似 `playwright/.auth` 的目錄，並加入 `.gitignore`，因為這類檔案可能包含敏感 cookie/token。

---

## Java Playwright 範例：使用已保存的登入狀態

```
PathauthStatePath=Paths.get("src/test/resources/e2e/auth-state.json");

BrowserContextcontext=browser.newContext(
newBrowser.NewContextOptions()
.setStorageStatePath(authStatePath)
.setAcceptDownloads(true)
);

Pagepage=context.newPage();

page.navigate("https://test-system.company.com/vendor/items");
```

然後測上傳：

```
page.locator("[data-testid='site-select']").selectOption("CY");

page.locator("[data-testid='vendor-upload-input']")
.setInputFiles(Paths.get("src/test/resources/e2e/vendor-upload-duplicate-name-cy.xlsx"));

page.locator("[data-testid='upload-button']").click();

page.locator("text=success").waitFor();
```

但這種做法有幾個問題：

```
1. auth-state.json 可能過期
2. cookie/token 是敏感資料，不可 commit
3. 公司 SSO 如果綁設備、IP、憑證、MFA，CI/CD 可能仍然不能用
4. 不同使用者角色需要不同 storage state
```

所以它適合本機或封閉測試環境，不一定適合內網 CI/CD。

   這層負責驗證上傳、下載、MinIO、vendor/site 對應邏輯。

2. UI E2E Test：
   若需要測畫面，用 Playwright Java 搭配 storage state 重用登入狀態，不在每支測試重複登入。

3. True SSO Smoke Test：
   只保留少量測試確認 TSSO / Keycloak redirect 流程正常，不把所有業務測試都綁在真實登入鏈路上。
```

這樣講是很工程化的，不是在逃避登入，而是在避免 E2E Test 變得不穩定。

# 11. 你現在最該做的具體清單

你下一步應該照這個順序處理：

```
1. 找出 TSSO / Keycloak 攔截點
   - Filter
   - Interceptor
   - session
   - UserContext
   - header parser

2. 確認是否可以用 property 關閉正式 auth
   - app.auth.enabled=false
   - tsso.enabled=false
   - security.enabled=false

3. 如果沒有開關，就補一個 ConditionalOnProperty

4. 在 test profile 注入 MockAuthFilter / MockUserInterceptor

5. REST Assured E2E Test 不登入，直接測上傳 / 下載 API

6. UI E2E 另開 profile，不跟後端 E2E 混在一起

7. 真正 Keycloak 登入只做 smoke test
```

# 12. 最重要的結論

你現在的痛苦是正常的，因為你把「業務 E2E」綁到「企業 SSO E2E」上了。

比較成熟的做法是：

> **業務流程測試不要每次真的登入；登入流程只做少數 smoke test。**
> 

你的 MinIO 上傳 / 下載 E2E Test，應該走：

```
SpringBootTest RANDOM_PORT
+ REST Assured
+ Testcontainers MinIO
+ e2e profile
+ MockAuthFilter / MockUserInterceptor
```

而不是：

```
Playwright 打開瀏覽器
→ TSSO
→ Keycloak
→ 公司內網跳轉
→ 登入成功
→ 再測上傳下載
```

後者可以做，但只能做少數，不應該成為你每次驗證 bug fix 的主要方式。
